너는 시니어 프론트엔드 개발자이자 UI/UX 설계 전문가다.

나는 "정보처리기사 학습 웹사이트"를 만들고 싶다.

[목표]
- 개인 학습 정리 + 복습 + 문제 생성까지 가능한 웹앱
- 완전 로컬 기반 (DB 없이 JSON 저장)
- 업데이트를 활용한 관리자 페이지 작성구조 [git에서 ignore로 업로드하지는 않을 페이지]

[기술 조건]
- HTML / CSS / JavaScript (Vanilla)
- 파일 구조 분리 (index.html / css / js)
- 로컬 스토리지 or JSON 기반 저장
- 유지보수 가능한 구조 (모듈화 필수)
- 관리자 페이지도 같이 메뉴에 추가 (실제 사용할때는 메뉴를 지워서 사용할 예정)
- 저장 방식은 JSON으로 관리자 페이지에서 저장된 내용을 JSON 또는 CSV 를 저장 불러오기가 가능하도록 해야함

[페이지 구성] (총 11개 - (HTML로 페이지를 나누어서))
1. 대시보드
2. 정보처리기사 필기 내용
3. 정보처리기사 실기 내용
4. SQL 문제 풀이 
5. 프로그래밍언어 문제 풀이
6. 문제 은행
7. 설정 페이지
8. [관리자 페이지] 블로그처럼 글쓰기와 같이 필기 내용을 작성하는 페이지
9. [관리자 페이지] (SQL, 프로그래밍언어) 문제를 추가, 풀이 내용을 작성하는 페이지
10. [관리자 페이지] 문제 은행에 문제를 추가하는 페이지
11. [관리자 페이지] 저장된 학습내용 전부 확인하기

[페이지별 핵심 역할]
1. 대시보드: 카드형태로 메뉴로 넘어가기(정보처리기사 필기 내용, 정보처리기사 실기 내용, SQL 문제풀이, 프로그래밍 언어 문제 풀이, 문제은행), 업데이트 내용
2. 정보처리기사 필기 내용: 과목/주제/소주제 기반 학습 콘텐츠 열람
3. 정보처리기사 실기 내용: 실기 학습 콘텐츠 열람
4. SQL 문제 풀이: SQL 문제, 해설, 정답, 분류별 열람
5. 프로그래밍언어 문제 풀이: 언어별 문제와 해설 학습
6. 문제 은행: 통합 문제 풀이 및 필터 검색
7. 설정 페이지: 테마, 데이터 import/export, 초기화
8. 관리자-필기 작성: 필기/실기 콘텐츠 작성 및 수정
9. 관리자-문제 작성: SQL/프로그래밍 문제 작성 및 수정
10. 관리자-문제은행 작성: 통합 문제은행 작성 및 수정
11. 관리자-전체 데이터 확인: 전체 저장 데이터 조회 및 검수, JSON 또는 CSV 파일 저장 및 불러오기

[문제페이지 구조]

[상단]
- 과목 선택 탭 (horizontal tabs)
- 스크롤 가능
- 활성화된 탭 강조

[중단]
- 선택된 과목의 하위 주제 표시
- grid 형태 (여러 줄)
- 버튼 클릭 시 콘텐츠 변경

[하단]
- 콘텐츠 표시 영역
- 카드형 UI
- 중앙 정렬
- 스크롤 가능

[동작 요구사항]

- 과목 탭 클릭 시:
  → 해당 과목의 부과목 목록이 동적으로 변경되어야 한다

- 부과목 클릭 시:
  → 해당 부과목의 내용이 메인 콘텐츠 영역에 표시되어야 한다

- 모든 데이터는 JSON 구조를 기반으로 렌더링한다
- DOM을 직접 수정하는 방식 (Vanilla JS)으로 구현한다
